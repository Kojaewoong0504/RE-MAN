# Recommendation Feedback Memory Design

## Goal

추천 반응 저장이 "저장 완료"에서 끝나지 않고, 다음 추천에 어떤 기준으로 반영되는지 사용자가 바로 알 수 있게 한다.

## Problem

현재 결과 화면은 반응 저장 후 "다음 스타일 체크에 반영합니다"만 보여준다. 실제로는 `preference_profile`에 좋아한 방향, 피할 방향, 메모가 구조화되어 전달되지만 UI에서는 그 기준이 보이지 않아 개인화가 약하게 느껴진다.

## Decisions

- 저장 후 결과 화면에 `다음 추천 기준` 요약을 보여준다.
- 업로드 화면의 `이전 반응 반영` 카드에도 같은 요약을 보여준다.
- 문구는 "모델 학습"이 아니라 "다음 체크 컨텍스트에 포함"으로 표현한다.
- 요약은 내부 `preference_profile` 규칙과 같은 source of truth를 사용한다.

## Acceptance Criteria

- `helpful` 반응은 `좋아한 방향`으로 표시된다.
- `not_sure` 반응은 `피할 방향`으로 표시된다.
- `save_for_later` 반응은 `보류 후보`로 표시된다.
- 메모가 있으면 `메모`로 표시된다.
- 결과 화면 저장 후 `다음 추천 기준`이 보인다.
- 업로드 화면에서 이전 반응이 있으면 `다음 추천 기준` 요약이 보인다.
