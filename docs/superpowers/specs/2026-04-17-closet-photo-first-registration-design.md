# Closet Photo-First Registration Design

## Goal

옷장 등록을 "폼 작성"이 아니라 "현실 옷장 사진을 먼저 저장하는 흐름"으로 바꾼다.

## Problem

현재 옷장 등록 모달은 사진 업로드가 가능하지만 이름, 색, 핏, 사이즈, 상태 입력이 한꺼번에 보여서 사용자가 먼저 무엇을 해야 하는지 흐려진다. MVP의 핵심은 옷장 사진과 카테고리를 빠르게 채워 다음 스타일 체크에 쓰는 것이다.

## Decisions

- 새 아이템 등록은 사진 미리보기와 사진 선택 CTA를 가장 크게 보여준다.
- 기본 입력은 `사진`과 `종류`만 보인다.
- 이름, 색, 핏, 사이즈, 착용감, 빈도, 계절, 상태, 메모는 `선택 정보 열기`를 눌렀을 때만 보인다.
- 사진 1장만 있으면 저장 가능하다. 이름이 없으면 기존처럼 `상의 사진`, `하의 사진` 같은 기본 이름을 사용한다.
- 수정 모드는 기존 정보를 바꿔야 하므로 선택 정보를 기본으로 펼친다.

## Acceptance Criteria

- 옷 추가 모달 첫 상태에서 `아이템 이름`, `색`, `핏`, `사이즈` 입력이 보이지 않는다.
- `선택 정보 열기`를 누르면 상세 입력이 보인다.
- 사진과 종류만으로 아이템을 저장할 수 있다.
- 상세 입력을 연 뒤 기존처럼 이름/색/사이즈/상태를 저장할 수 있다.
- 전체 옷장 저장 후 다음 분석 컨텍스트에 `closet_items`가 유지된다.
