# Result Density Reduction Design

## Goal

스타일 체크 결과 첫 화면을 "한 번에 읽히는 실행 화면"으로 줄인다.

## Visual Thesis

차분한 앱 표면 위에 추천 조합과 오늘 행동을 먼저 세우고, 상세 설명은 사용자가 필요할 때만 여는 구조로 둔다.

## Content Plan

1. 상단: 추천 조합명, 3개 아이템, 오늘 할 일
2. 근거 요약: 상의/하의/신발이 어떤 옷장에서 왔는지 짧은 칩으로 표시
3. 상세: 진단/추천 이유, 근거 상세, 개선 포인트, 조합 느낌, 사이즈 후보, 반응 저장, 계정 저장
4. 최종 행동: 새 체크, 옷장, 기록

## Interaction Thesis

- 첫 화면에서는 읽을 문장을 줄이고, 사용자가 눌렀을 때 상세를 펼친다.
- 옷장 근거는 기본 상태에서 칩 3개만 보이고, 상세 카드의 착용감/상태는 펼침 후에 보인다.
- 대기 시간이 있는 기능은 기본 화면에서 호출되지 않는다.

## Decisions

- 진단과 추천 이유는 기본 화면에서 숨긴다.
- 추천 조합 아이템과 오늘 할 일은 기본 화면에 남긴다.
- 옷장 근거는 기본 화면에서 `카테고리`, `아이템명`, `추천에 사용/후보`만 보여준다.
- `자주 입고 잘 맞음`, `핏/상태 확인`, 사이즈/착용감 상세는 `근거 자세히 보기`를 눌렀을 때만 보여준다.
- 기존 보조 기능은 제거하지 않고 접힌 상태로 유지한다.

## Acceptance Criteria

- 결과 화면 첫 진입에서 진단 문장과 추천 이유 문장이 보이지 않는다.
- `진단과 이유 보기`를 누르면 진단과 추천 이유가 보인다.
- 결과 화면 첫 진입에서 근거 상세 신호 문구가 보이지 않는다.
- `근거 자세히 보기`를 누르면 근거 상세 카드와 신호 문구가 보인다.
- `새 체크`, `옷장`, `기록` 액션은 유지된다.
- `npm run visual:app` 산출물에서 모바일 결과 화면의 첫 화면이 요약 중심으로 보인다.
