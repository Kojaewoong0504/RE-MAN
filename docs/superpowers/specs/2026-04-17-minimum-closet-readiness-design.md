# Minimum Closet Readiness Design

## Purpose

스타일 추천은 사용자의 현실 옷장을 기준으로 해야 하므로, 최소한 상의, 하의, 신발 컨텍스트가 있어야 한다.
옷장 아이템이 1개만 있는 상태에서 분석을 허용하면 추천 품질과 사용자의 신뢰가 떨어진다.

## Scope

- 필수 카테고리는 `tops`, `bottoms`, `shoes`다.
- `outerwear`는 선택 항목이다.
- 업로드 화면은 필수 카테고리별 준비 상태를 보여준다.
- 옷장 화면은 부족한 필수 카테고리를 짧게 안내한다.
- 분석 시작 버튼은 사진/텍스트 입력과 필수 옷장 카테고리가 모두 준비됐을 때만 활성화한다.

## Non-Goals

- 카테고리별 권장 개수 2개 이상은 요구하지 않는다.
- 계절별 옷장 완성도는 계산하지 않는다.
- 옷장 사진 자동 판독은 하지 않는다.

## Verification

- 단위 테스트는 필수 카테고리 준비 상태 계산을 검증한다.
- E2E는 상의만 등록한 업로드 화면에서 분석 버튼이 비활성화되고, 부족한 하의/신발 안내가 보이는지 검증한다.
- 기존 전체 플로우 E2E는 상의/하의/신발을 모두 등록하므로 계속 통과해야 한다.
