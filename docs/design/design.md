# DESIGN.md

## 개요

이 문서는 모든 화면 설계의 기준이다.
디자이너와 프론트엔드 개발자가 애매한 결정을 할 때 여기서 답을 찾는다.
core-beliefs.md의 3가지 신념이 디자인에도 그대로 적용된다.

## 비주얼 무드 — Warm Utility Masculine

> "사진을 기준으로 차분하게 진단하고, 오늘 바꿀 한 가지를 또렷하게 보여준다."

이 앱은 너드남이 처음으로 "지금 가진 옷으로 시작할 수 있다"는 감각을 갖게 만든다.
그 감각은 따뜻한 스톤 배경, 실제 사진 중심의 큰 시각 앵커, 단단한 블랙 타이포그래피, 브라스/올리브 계열 CTA에서 온다.

포스터형 강한 보더와 밝은 노란색 포인트는 쓰지 않는다.
강조색은 남성적인 브라스/올리브 계열로 제한한다.
기본 화면은 실제 앱처럼 차분하고 읽기 쉬워야 한다.

### 컬러 팔레트

| 용도 | 색상 | 비고 |
|---|---|---|
| Background | `--color-bg: #F2EEE3` | 앱 배경 기본 |
| Background Soft | `--color-bg-soft: #E7DFD0` | 화면 하단/보조 배경 |
| Surface | `--color-surface: #DDD4C2` | 보조 패널 |
| Surface Raised | `--color-surface-raised: #FAF6ED` | 기본 패널/입력 표면 |
| Accent | `--color-accent: #6F5F2D` | 브라스/올리브 계열 CTA 또는 핵심 미션 1개 |
| Accent Ink | `--color-accent-ink: #F9F2DF` | 강조색 위 텍스트 |
| Text Primary | `--color-ink: #15130F` | 본문 |
| Text Secondary | `--color-muted: #6B6558` | 보조 텍스트 |
| Line | `--color-line: rgba(21,19,15,0.16)` | 기본 구분선 |
| Success | `#4ADE80` | 미션 완료 |
| Error | `#F87171` | 실패/경고 |

색상은 `app/globals.css`의 CSS variable을 기준으로 관리한다.
Tailwind theme 색상도 이 variable을 참조한다.

### 타이포그래피

- **폰트**: Pretendard (한글), Inter (영문)
- **헤드라인**: 32px / Bold / 자간 -0.5px
- **서브헤드**: 20px / SemiBold
- **본문**: 16px / Regular / 행간 1.6
- **캡션**: 13px / Regular / `#888888`

### 기타 스타일 규칙

- 보더: 기본 `1px` 구분선. `2px` 포스터 보더는 새 컴포넌트에서 기본값으로 쓰지 않는다.
- 보더 라디우스: 제품 UI에서는 기본적으로 쓰지 않는다. 계정 아바타처럼 의미가 있는 요소에만 제한적으로 사용한다.
- 그림자: 사용하지 않는다. 면 대비와 타이포그래피로 위계를 만든다.
- 아이콘: 선형(outline) 아이콘만 사용, filled 금지
- 이미지 필터: 유저 사진 외 스톡 이미지 사용 금지

### 디자인 시스템 v1

공통 UI는 직접 색상/보더를 반복하지 않고 아래 클래스를 우선 사용한다.

| 클래스 | 용도 |
|---|---|
| `.app-shell` | 모바일 앱 셸, 최대 480px, 하단 CTA 여백 포함 |
| `.app-header` | 상단 헤더와 1px 하단 구분선 |
| `.app-brand`, `.app-subbrand` | 브랜드/보조 브랜드 라벨 |
| `.ui-section` | 화면 안의 주요 섹션 구분 |
| `.ui-panel`, `.ui-panel-muted`, `.ui-panel-accent` | 기본/보조/강조 패널 |
| `.ui-button`, `.ui-button-secondary`, `.ui-button-accent` | 기본/보조/강조 버튼 |
| `.ui-choice`, `.ui-choice-selected` | 설문/선택 버튼 |
| `.ui-input` | 단일 행 입력과 셀렉트 |

새 화면에서 `border-2`, `bg-accent`, `bg-[#fcf8ef]`를 직접 반복하지 않는다.
필요한 경우 먼저 디자인 토큰 또는 공통 클래스를 확장한다.

## 레이아웃 원칙

### 기본 구조

- 모바일 우선 설계 (max-width: 480px 기준)
- 단일 컬럼 레이아웃
- 화면당 핵심 액션 1개만 — CTA는 항상 하단 고정

### 피드백 결과 레이아웃

AI 피드백은 한 화면에서 바로 이해되어야 한다.
추천 조합과 오늘 미션을 카드 넘김 안에 숨기지 않는다.

```
[업로드한 사진]
[진단 한 줄]
[개선 포인트 3개]
[추천 조합]
[오늘의 액션 1개]
[레퍼런스 보기 또는 실착 생성]
```

카드는 카드 자체가 상호작용일 때만 사용한다.

## 화면별 디자인 원칙

### 랜딩
- 헤드카피가 화면의 60% 차지
- CTA 버튼은 브라스/올리브 Accent 컬러
- 따뜻한 스톤 배경, 카드 남용 금지
- 배경 이미지 없음 — 텍스트와 면 분할만으로 무게감

### 설문
- 질문은 세 개까지 한 흐름에서 노출할 수 있다.
- 선택지는 큰 탭 버튼 형태이며 선택 상태는 `ui-choice-selected`로 표시한다.
- 진행 바보다 현재 단계와 왜 묻는지 설명하는 텍스트를 우선한다.

### 사진 업로드
- 업로드 영역은 큰 사진 프레임을 먼저 보여준다.
- "잘 나올 필요 없어요" 카피 강조
- 가이드 예시는 작게, 방해되지 않게

### 로딩
- 밝은 배경에 잉크색 텍스트 애니메이션
- 프로그레스 바 없음 — 텍스트 전환으로 진행감 표현

### 피드백 결과
- 업로드한 사진을 기준 이미지로 유지
- 진단, 개선 포인트, 추천 조합, 오늘 액션을 한 화면에서 이해 가능하게 구성
- 실착 영역은 `mock` 레퍼런스와 실제 Vertex 생성을 명확히 분리
- 하단 고정 CTA: "결과 저장하고 홈으로"
- 루틴 모드는 보조 링크로만 제공
