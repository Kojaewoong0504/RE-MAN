## 비주얼 무드 — Editorial Coach App

> "사진을 기준으로 차분하게 진단하고, 오늘 바꿀 한 가지를 또렷하게 보여준다."

이 앱은 너드남이 처음으로 "지금 가진 옷으로 시작할 수 있다"는 감각을 갖게 만든다.
그 감각은 밝은 바탕, 실제 사진 중심의 큰 시각 앵커, 절제된 블랙 타이포그래피, 명확한 CTA에서 온다.

포스터형 강한 보더와 네온 포인트는 강조 구간에서만 제한적으로 사용한다.
기본 화면은 실제 앱처럼 차분하고 읽기 쉬워야 한다.

### 컬러 팔레트

| 용도 | 색상 | 비고 |
|---|---|---|
| Primary | `#FCF8EF` (웜 화이트) | 배경 기본 |
| Accent | `#F2E94E` (포스터 옐로우) | 핵심 CTA 1개 |
| Surface | `#F4ECDD` | 카드/패널 배경 |
| Text Primary | `#111111` | 본문 |
| Text Secondary | `#6B6558` | 보조 텍스트 |
| Success | `#4ADE80` | 미션 완료 |
| Error | `#F87171` | 실패/경고 |

### 타이포그래피

- **폰트**: Pretendard (한글), Inter (영문)
- **헤드라인**: 32px / Bold / 자간 -0.5px
- **서브헤드**: 20px / SemiBold
- **본문**: 16px / Regular / 행간 1.6
- **캡션**: 13px / Regular / `#888888`

### 기타 스타일 규칙

- 보더 라디우스: 12px (카드), 8px (버튼)
- 그림자: 최소화 — 종이 겹침 같은 면 대비 우선
- 아이콘: 선형(outline) 아이콘만 사용, filled 금지
- 이미지 필터: 유저 사진 외 스톡 이미지 사용 금지

## 레이아웃 원칙

### 기본 구조

- 모바일 우선 설계 (max-width: 480px 기준)
- 단일 컬럼 레이아웃
- 화면당 핵심 액션 1개만 — CTA는 항상 하단 고정

### 피드백 결과 레이아웃

AI 피드백은 한 화면에서 바로 이해되어야 한다.
추천 조합과 오늘 미션을 카드 넘김 안에 숨기지 않는다.

```
[업로드한 사진]
[진단 한 줄]
[개선 포인트 3개]
[추천 조합]
[오늘의 액션 1개]
[레퍼런스 보기 또는 실착 생성]
```

카드는 카드 자체가 상호작용일 때만 사용한다.

## 화면별 디자인 원칙

### 랜딩
- 헤드카피가 화면의 60% 차지
- CTA 버튼은 Accent 컬러
- 밝은 포스터형 배경, 카드 남용 금지
- 배경 이미지 없음 — 텍스트와 면 분할만으로 무게감

### 설문
- 질문 1개씩 풀스크린 노출 (한 화면에 여러 질문 금지)
- 선택지는 큰 탭 버튼 형태
- 진행 바 상단 표시 (1/3 → 2/3 → 3/3)

### 사진 업로드
- 업로드 영역 중앙 배치, 점선 테두리
- "잘 나올 필요 없어요" 카피 강조
- 가이드 예시는 작게, 방해되지 않게

### 로딩
- 밝은 배경에 잉크색 텍스트 애니메이션
- 프로그레스 바 없음 — 텍스트 전환으로 진행감 표현

### 피드백 결과
- 업로드한 사진을 기준 이미지로 유지
- 진단, 개선 포인트, 추천 조합, 오늘 액션을 한 화면에서 이해 가능하게 구성
- 실착 영역은 `mock` 레퍼런스와 실제 Vertex 생성을 명확히 분리
- 하단 고정 CTA: "Day 1 미션 시작하기"
