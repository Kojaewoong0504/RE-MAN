# PRODUCT_SENSE.md

## 1. Service Definition

RE:MAN은 너드남이 판단 없이 자기 변화를 시작하고 이어갈 수 있게 돕는 AI 변화 코치 서비스다.

핵심은 "완성된 사람처럼 보이게 만들기"가 아니라 "지금 상태에서 첫 변화를 실제로 시작하게 만드는 것"이다.

스타일은 첫 번째 프로그램일 뿐이며, 장기적으로는 헤어, 체형/자세, 피부까지 확장한다.

---

## 2. Problem

스타일을 바꾸고 싶은 2030 너드남은 많다.
하지만 대부분 행동으로 옮기지 못한다.

이유는 정보가 없어서가 아니다.
유튜브에 스타일 영상은 넘쳐난다.

문제는 **시작점이 없기 때문**이다.

- 어떻게 입어야 할지 몰라서 그냥 안 함
- 쇼핑 자체가 귀찮고 어디서 시작할지 막막함
- 주변에 편하게 물어볼 사람이 없음
- 바꾸고 싶은 의지는 있는데 첫 발을 못 뗌

---

## 3. Insight

기존 해결책(유튜브, 블로그, 무신사 코디)의 실패 이유는 두 가지다.

1. **나의 현재 상태를 반영하지 않는다** — 일반론이지, 내 얘기가 아님
2. **판단받는 느낌이 든다** — 사람한테 물어보는 건 창피함

AI 코칭이 다른 이유:
- 내 사진을 직접 보고 말해주니까 → 신뢰감이 다름
- 판단받을 걱정이 없으니까 → 솔직하게 시작할 수 있음

핵심 인사이트:
> "판단받지 않는 환경 + 내 상황 기반 피드백 = 처음으로 시작할 수 있는 조건"

---

## 4. Target User

**메인 페르소나: 김지훈 (29세)**

- 테크 기업 백엔드 개발자
- 매일 회사에서 받은 AWS/Google 티셔츠 + 청바지 + 뉴발란스 990
- 패션 유튜브를 가끔 보긴 하는데 "나한테 해당 없는 얘기"로 느낌
- 소개팅이나 커리어 전환 같은 계기가 생기면 "좀 바꿔야 하나" 생각함
- 하지만 뭘 사야 할지, 어디서 시작할지 몰라서 결국 안 함

**공통 특성:**
- 2030 남성
- 평소 자신을 꾸미는 것에 관심 없었음
- 변화의 의지는 있으나 첫 행동을 못 함
- 판단받는 상황을 불편해함

---

## 5. Core Value Proposition

> "판단 없이, 지금 상태에서 시작하는 AI 변화 코칭"

- **판단 없이**: AI니까 창피하지 않음
- **지금 상태에서**: 내 현재 모습을 기준으로 시작함
- **작게 시작**: 한 번에 하나만 바꾸게 함
- **확장 가능**: 스타일에서 시작해 다른 변화 영역으로 이어질 수 있음

---

## 6. Information Architecture Principle

앱 첫 진입은 설문이 아니라 `랜딩 + 기능 선택`에서 시작한다.

- 처음 온 사용자는 먼저 서비스와 변화 영역을 이해해야 한다.
- 재방문 사용자는 매번 처음 루트를 다시 타면 안 된다.
- 기능별 설문과 업로드는 해당 프로그램을 고른 뒤에만 들어간다.

초기 기능 허브 구성:

- 스타일: 바로 시작 가능
- 헤어: 준비 중
- 체형/자세: 준비 중
- 피부: 준비 중

스타일은 앱 전체 onboarding이 아니라 첫 번째 프로그램이다.

---

## 7. User State Principle

사용자 상태는 최소 아래 3가지로 구분한다.

1. `new_visitor`
- 아직 어떤 프로그램도 시작하지 않은 사용자
- 랜딩과 기능 허브를 먼저 본다

2. `active_program_user`
- 예: 스타일 Day 3 진행 중
- 홈 진입 시 현재 프로그램으로 바로 복귀한다

3. `completed_program_user`
- 예: 스타일 7일 완료
- 완료 요약을 보고, 같은 프로그램을 다시 보거나 다른 프로그램을 고른다

---

## 8. What We're NOT (MVP 범위 외)

다음은 의도적으로 만들지 않는다:

- ❌ 소셜 공유 / 커뮤니티 기능 — 판단받는 환경을 만들지 않음
- ❌ 유명 브랜드 위주 콘텐츠 — 접근 가능한 스타일이 우선
- ❌ 여성 코칭 — 페르소나 집중
- ❌ 여러 프로그램을 한 번에 동시에 시작하게 하는 구조
- ❌ 1주 이후 유료 구독 — MVP는 전환 경험 자체가 목표
