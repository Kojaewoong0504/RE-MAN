# Coaching Flow

## Overview

RE:MAN의 기본 코칭 플로우는 `Style Check Session`이다.
사용자는 하루 안에 사진 분석, 추천 조합, 다음 행동을 확인할 수 있어야 한다.

기존 `Day 1~7` 루틴은 기본 플로우가 아니라 선택형 `Routine Mode`다.

---

## Default Session Loop

```text
[입력] 사진 또는 텍스트 + 옷장 스냅샷 + 목표/자신감
  ↓
[진단] 현재 스타일의 시작점 확인
  ↓
[추천] 지금 가진 옷 기준 추천 조합 1개
  ↓
[저장] 추천 반응과 결과 저장
  ↓
[반영] 다음 체크에 이전 반응을 컨텍스트로 포함
  ↓
[선택] 옷장 수정, 기록 보기, 새 체크 또는 홈 복귀
```

원칙:

- 세션 하나에서 핵심 가치를 모두 보여준다.
- 사용자가 매일 방문해야만 좋아지는 구조를 기본값으로 두지 않는다.
- 결과 화면은 긴 순차 카드가 아니라 한 번에 스캔 가능한 구조여야 한다.
- 저장한 추천 반응은 다음 체크에서 반영된다는 점이 UI에서 보여야 한다.
- deep dive, 실착, 크레딧처럼 대기와 비용이 있는 기능은 MVP 기본 화면에서 숨긴다.
- 구매 유도는 사용자가 명시적으로 구매 후보를 보고 싶다고 선택하기 전까지 하지 않는다.

---

## Optional Deep Dive Modules

세션 결과 이후의 확장 후보:

- `fit-check`: 핏, 기장, 실루엣
- `color-check`: 색 조합과 톤
- `occasion-check`: 소개팅, 출근, 주말, 면접 같은 상황별 코디
- `closet-remix`: 현재 옷장 기준 다른 조합
- `try-on`: 레퍼런스 또는 사용자가 올린 상품 이미지 실착
- `shopping-candidate`: 부족한 아이템 후보. 사용자가 명시적으로 요청할 때만 제공

MVP에서는 deep dive를 모두 구현하지 않아도 된다.
중요한 것은 기본 제품 모델이 `일일 숙제`가 아니라 `선택형 추가 체크`라는 점이다.

---

## Routine Mode

루틴 모드는 아래 조건을 만족할 때만 노출한다.

- 사용자가 결과를 본 뒤 직접 선택한다.
- 루틴을 건너뛰거나 하루에 여러 번 진행해도 막지 않는다.
- 루틴 미완료 패널티나 독촉을 만들지 않는다.
- 루틴 화면은 legacy 경로로 남길 수 있으나 홈/결과의 기본 CTA가 되면 안 된다.

---

## Success Metrics

- 스타일 체크 결과 도달율
- 결과 화면 체류와 CTA 클릭률
- 추천 조합/레퍼런스 영역 사용률
- 추가 체크 선택률
- 재방문 시 최근 결과 복귀율
