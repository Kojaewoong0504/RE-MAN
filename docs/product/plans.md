# PLANS.md

## 목표

**MVP를 2~3주 안에 출시한다.**
범위는 최소화, 유저가 Day 1 피드백까지 받는 경험을 완성하는 것이 1차 목표.
Day 2~7 플로우는 MVP 이후 단계에서 완성한다.

---

## 마일스톤

### Phase 0 — 세팅 (Day 1~2)
- [ ] 레포 세팅 및 기술 스택 확정 (ARCHITECTURE.md 참고)
- [ ] Gemini API 연동 테스트
- [ ] 사진 업로드 → AI 응답 최소 파이프라인 확인
- [ ] 배포 환경 세팅 (도메인, 호스팅)

**완료 기준**: 사진 1장 보내면 Gemini가 JSON 응답을 돌려주는 것

---

### Phase 1 — 온보딩 완성 (Day 3~7)

유저가 랜딩에서 AI 첫 피드백까지 끊김 없이 도달하는 것.

- [ ] 랜딩 페이지 (헤드카피 + CTA)
- [ ] 설문 3개 화면
- [ ] 사진 업로드 화면 (가이드 이미지 포함)
- [ ] AI 분석 중 로딩 화면
- [ ] 첫 AI 피드백 결과 화면
- [ ] 이메일 수집 화면 (선택)

**완료 기준**: 실제 사람 5명이 랜딩 → 첫 피드백까지 완주

---

### Phase 2 — Day 2~7 플로우 (Day 8~14)

- [ ] 매일 미션 노출 화면
- [ ] 사진 업로드 → daily-agent 피드백 화면
- [ ] 피드백 이력 저장 및 누적 전달
- [ ] Day 7 Before/After 비교 화면
- [ ] 이메일로 다음 날 미션 발송 (선택)

**완료 기준**: Day 1~7 전체 플로우를 한 명이 완주

---

### Phase 3 — 안정화 (Day 15~21)

- [ ] 모바일 반응형 UI 점검
- [ ] AI 응답 실패 시 fallback 처리
- [ ] 사진 업로드 용량/포맷 제한 처리
- [ ] 기본 에러 페이지
- [ ] 간단한 사용 지표 수집 (완료율, 이탈 지점)

**완료 기준**: 버그 없이 10명 연속 완주 가능

---

## 개발 우선순위 원칙

1. **플로우 먼저, 디자인 나중** — 못생겨도 동작하는 게 우선
2. **AI 파이프라인이 막히면 전체가 막힌다** — Phase 0에서 반드시 검증
3. **Day 2~7보다 Day 1이 먼저** — 첫 경험이 좋아야 Day 2가 의미 있음
4. **이메일 기능은 마지막** — 없어도 MVP는 동작한다
