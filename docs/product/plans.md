# Plans

## 목표

**MVP를 2~3주 안에 출시한다.**
범위는 최소화하고, 유저가 하루 안에 `Style Check Session` 결과까지 받는 경험을 완성하는 것이 1차 목표다.

기본 플로우는 7일 루틴이 아니다.
루틴은 사용자가 원할 때 선택하는 보조 모드로 둔다.

---

## 마일스톤

### Phase 0 — 세팅

- [ ] 레포 세팅 및 기술 스택 확정
- [ ] Gemini API 연동 테스트
- [ ] 사진 업로드 → AI 응답 최소 파이프라인 확인
- [ ] 배포 환경 세팅

**완료 기준**: 사진 1장 또는 텍스트 설명을 보내면 Gemini가 계약에 맞는 JSON 응답을 돌려주는 것

---

### Phase 1 — Style Check Session 완성

유저가 랜딩에서 스타일 체크 결과까지 끊김 없이 도달하는 것.

- [ ] 랜딩 페이지
- [ ] 프로그램 허브
- [ ] 설문 3개
- [ ] 사진/텍스트 입력
- [ ] 옷장 스냅샷
- [ ] 목표/자신감 선택
- [ ] AI 분석 중 로딩 화면
- [ ] 세션 결과 화면
- [ ] 결과 다음 행동 허브
- [ ] 추천 조합 레퍼런스
- [ ] 실착 provider 비활성/활성 상태 분리

**완료 기준**: 실제 사람 5명이 랜딩 → 스타일 체크 결과까지 완주

---

### Phase 2 — 추가 체크 모듈

세션 결과 이후 사용자가 선택할 수 있는 deep dive를 만든다.

- [x] 핏 더 보기
- [x] 색 조합 더 보기
- [x] 상황별 코디
- [x] 내 옷장 기준 다른 조합
- [ ] 실착 미리보기
- [ ] 구매 후보는 사용자가 명시적으로 원할 때만 제공

**완료 기준**: 세션 결과를 본 사용자가 추가 체크를 선택하고 다시 결과를 받을 수 있음

---

### Phase 3 — 선택형 루틴 모드

루틴은 기본 플로우가 아니라 선택 경로다.

- [ ] 루틴 시작 여부를 명시적으로 선택
- [ ] 하루에 여러 체크를 진행해도 막지 않음
- [ ] 루틴 미완료 패널티/독촉 없음
- [ ] 기존 daily-agent 경로는 legacy/optional로 유지

**완료 기준**: 기본 CTA가 루틴으로 이어지지 않고, 사용자가 선택할 때만 루틴에 진입

---

### Phase 4 — 안정화

- [ ] 모바일 반응형 UI 점검
- [ ] AI 응답 실패 fallback 처리
- [ ] 사진 업로드 용량/포맷 제한 처리
- [ ] 기본 에러 페이지
- [ ] 간단한 사용 지표 수집

**완료 기준**: 버그 없이 10명 연속 스타일 체크 결과까지 완주 가능

---

## 개발 우선순위 원칙

1. **세션 결과 먼저** — 7일 루틴보다 하루 안에 핵심 가치를 보여주는 것이 우선
2. **AI 파이프라인이 막히면 전체가 막힌다** — 먼저 계약과 fallback을 검증
3. **추가 체크는 선택형** — 사용자의 속도를 앱이 강제하지 않는다
4. **이메일/알림 기능은 마지막** — 없어도 MVP는 동작한다
