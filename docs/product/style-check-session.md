# Style Check Session

## Purpose

스타일 프로그램의 기본 단위는 `7일 챌린지`가 아니라 `Style Check Session`이다.
사용자는 하루 안에 현재 상태, 추천 조합, 레퍼런스/실착 가능성을 모두 확인할 수 있어야 한다.

7일 루틴은 기본 경로가 아니다.
꾸준히 해보고 싶은 사용자가 선택하는 보조 모드로 둔다.

---

## Default Flow

```text
랜딩
  -> 프로그램 허브
  -> 스타일 체크 소개
  -> 설문 3개
  -> 사진 또는 텍스트 + 옷장 스냅샷 + 목표/자신감
  -> 분석
  -> 세션 결과
```

세션 결과에서 반드시 보여줄 것:

- 현재 사진/설명 기반 진단
- 개선 포인트 3개
- 지금 가진 옷 기준 추천 조합 1개
- 추천 조합 이유
- 레퍼런스 또는 실착 미리보기 영역
- 바로 할 수 있는 행동 1개
- 선택형 다음 행동 허브

다음 행동 허브의 MVP 항목:

- 핏 더 보기
- 색 조합 보기
- 상황별 코디
- 내 옷장 다른 조합
- 새 스타일 체크 시작하기

아직 구현되지 않은 deep dive는 완료된 기능처럼 보여주지 않는다.
준비 중 안내를 노출하고, API가 연결되기 전에는 기존 결과 기반 placeholder 이상으로 동작하지 않는다.
`새 스타일 체크 시작하기`는 기존 결과와 사진 입력을 비우고, 설문/옷장 컨텍스트는 유지한 채 업로드 화면으로 돌아간다.

MVP 구현 상태:

- `핏 더 보기`: `/api/deep-dive`의 `module: "fit"`로 실제 응답을 생성한다.
- `색 조합 보기`: `/api/deep-dive`의 `module: "color"`로 실제 응답을 생성한다.
- `상황별 코디`: `/api/deep-dive`의 `module: "occasion"`으로 실제 응답을 생성한다.
- `내 옷장 다른 조합`: `/api/deep-dive`의 `module: "closet"`으로 실제 응답을 생성한다.

세션 결과는 한 번에 이해되어야 한다.
사용자가 `Day 1`, `Day 2` 같은 순차 루프를 따라야만 핵심 가치를 확인할 수 있으면 실패다.

---

## Optional Deep Dives

세션 결과 이후 사용자가 선택할 수 있는 다음 행동:

- 핏 더 자세히 보기
- 색 조합 더 보기
- 상황별 코디로 다시 보기
- 내 옷장 기준으로 다른 조합 보기
- 실착 미리보기 만들기
- 새 스타일 체크 시작하기

MVP에서는 모든 deep dive를 즉시 구현하지 않아도 된다.
다만 UI와 문서에서는 `7일 루틴 강제`가 아니라 `선택형 추가 체크`가 기본 방향이어야 한다.

---

## Routine Mode

기존 `Day 1~7` 루프는 삭제하지 않고 `Routine Mode`로 분리한다.

원칙:

- 기본 CTA가 루틴으로 이어지면 안 된다.
- 루틴은 사용자가 명시적으로 선택할 때만 시작한다.
- 매일 방문을 전제로 하지 않는다.
- 하루에 여러 체크를 진행해도 막지 않는다.
- 구매 유도는 사용자가 명시적으로 구매 후보를 보고 싶다고 할 때만 허용한다.

---

## Revisit Rule

재방문 사용자는 `오늘은 Day 3입니다`가 아니라 `최근 스타일 체크 결과`로 복귀한다.

홈 상태:

- `new`: 아직 스타일 체크 결과 없음
- `active`: 최근 스타일 체크 결과 있음
- `completed`: 루틴을 완료한 legacy 상태가 있더라도 기본 진입은 최근 스타일 체크 결과

기본 진입 경로는 `/programs/style/onboarding/result`다.
