# Style Check Session

## Purpose

스타일 프로그램의 기본 단위는 `7일 챌린지`가 아니라 `Style Check Session`이다.
사용자는 하루 안에 현재 상태, 추천 조합, 오늘 바로 할 행동을 확인할 수 있어야 한다.

7일 루틴은 기본 경로가 아니다.
꾸준히 해보고 싶은 사용자가 선택하는 보조 모드로 둔다.

---

## Default Flow

```text
랜딩
  -> 프로그램 허브
  -> 스타일 체크 소개
  -> 설문 3개
  -> 사진 또는 텍스트 + 옷장 스냅샷 + 목표/자신감
  -> 분석
  -> 세션 결과
```

세션 결과에서 반드시 보여줄 것:

- 현재 사진/설명 기반 진단
- 개선 포인트 3개
- 지금 가진 옷 기준 추천 조합 1개
- 추천 조합이 사용자의 옷장 중 어떤 아이템을 근거로 삼았는지
- 추천 조합 이유
- 바로 할 수 있는 행동 1개
- 추천 반응 저장
- 옷장 수정, 기록 보기, 새 스타일 체크 시작

다음 행동 허브의 MVP 항목:

- 옷장 수정하기
- 저장 기록 보기
- 새 스타일 체크 시작하기

deep dive, 실제 실착 생성, 크레딧 구매, 사이즈 후보, 내부 상품 후보는 MVP 기본 결과 화면에 노출하지 않는다.
대기 시간이 있는 생성 기능 버튼이 동시에 여러 개 보이면 실패다.
`새 스타일 체크 시작하기`는 기존 결과와 사진 입력을 비우고, 설문/옷장 컨텍스트는 유지한 채 업로드 화면으로 돌아간다.

결과 화면 UX 규칙:

- 추천 조합과 오늘 행동을 화면 상단에서 즉시 볼 수 있어야 한다.
- 진단 문장과 추천 이유 문장은 첫 화면에서 숨기고 `진단과 이유 보기`를 눌렀을 때만 보여준다.
- 옷장 근거는 추천 조합 바로 아래에서 칩 요약으로 보여준다.
- 옷장 근거는 `source_item_ids`가 있으면 `추천에 사용`, 없으면 보수적 후보 매칭으로 `비슷한 후보` 또는 `추가 후보`로 표시한다.
- 옷장 근거 상세는 `근거 자세히 보기`를 눌렀을 때만 보여주고, 상세 안에서 `자주 입고 잘 맞음`, `핏/상태 확인`, `후보`처럼 짧은 신호 문구를 붙인다.
- 옷장 근거를 상품 판정이나 구매 추천처럼 표현하지 않는다.
- 결과 화면: 진단, 추천 조합, 옷장 근거, 오늘 행동, 개선 포인트 토글, 조합 느낌 보기, 추천 반응 저장, 계정 저장, 옷장/기록/새 체크 연결만 제공한다.
- 결과 첫 화면은 추천 조합, 옷장 근거, 오늘 행동, 다음 액션만 먼저 보여준다.
- `조합 느낌 보기`는 생성 없는 레퍼런스/준비 패널이다. `/api/try-on` 호출, provider 호출, 크레딧 차감이 발생하면 실패다.
- 사이즈 후보는 기본 화면을 밀어내지 않도록 접힌 보조 패널로 제공한다.
- 추천 반응 저장과 계정 저장은 필요할 때 여는 보조 패널로 둔다. 기본 화면에 긴 폼을 항상 펼쳐두지 않는다.
- `/api/deep-dive`, `/api/try-on`, `/api/credits` 계약은 남아 있어도 MVP 프론트에서는 호출 진입점을 제공하지 않는다.

세션 결과는 한 번에 이해되어야 한다.
사용자가 `Day 1`, `Day 2` 같은 순차 루프를 따라야만 핵심 가치를 확인할 수 있으면 실패다.

---

## Deferred Deep Dives

세션 결과 이후 사용자가 선택할 수 있는 다음 행동:

- 핏 더 자세히 보기
- 색 조합 더 보기
- 상황별 코디로 다시 보기
- 내 옷장 기준으로 다른 조합 보기
- 실착 미리보기 만들기
- 새 스타일 체크 시작하기

MVP에서는 위 deep dive를 기본 프론트에 노출하지 않는다.
다시 노출하려면 문서 승인, 중복 요청 방지, 대기 중 비활성화, E2E 검증이 먼저 필요하다.

---

## Routine Mode

기존 `Day 1~7` 루프는 삭제하지 않고 `Routine Mode`로 분리한다.

원칙:

- 기본 CTA가 루틴으로 이어지면 안 된다.
- 루틴은 사용자가 명시적으로 선택할 때만 시작한다.
- 매일 방문을 전제로 하지 않는다.
- 하루에 여러 체크를 진행해도 막지 않는다.
- 구매 유도는 사용자가 명시적으로 구매 후보를 보고 싶다고 할 때만 허용한다.

---

## Revisit Rule

재방문 사용자는 `오늘은 Day 3입니다`나 `최근 스타일 체크 결과`로 강제 복귀하지 않는다.
홈과 스타일 시작 화면은 새 체크를 시작하는 진입점으로 유지한다.

홈 상태:

- `new`: 아직 스타일 체크 결과 없음
- `active`: 스타일 체크 결과가 있어도 기본 CTA는 새 체크
- `completed`: 루틴을 완료한 legacy 상태가 있어도 기본 CTA는 새 체크

기본 진입 경로는 `/programs/style/onboarding/upload?reset=photo`다.
과거 결과는 `/history`에서만 찾아본다.
