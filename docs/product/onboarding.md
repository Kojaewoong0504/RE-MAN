# new-user-onboarding.md

## Overview

이 문서는 앱 전체 onboarding이 아니라 `스타일 프로그램 onboarding`을 정의한다.
즉 사용자가 랜딩과 기능 선택 허브를 지난 뒤, 스타일 프로그램을 시작했을 때만 이 플로우를 탄다.

목표:
- **5분 안에 스타일 체크 결과까지 도달**
- 이탈 방지
- 프로그램 단위 진입 구조 유지
- 사용자가 7일 루프를 타지 않아도 핵심 가치를 하루 안에 확인

---

## Flow

### App Entry — 랜딩 + 기능 허브

- 앱 첫 화면은 설문이 아니다
- 사용자는 먼저 랜딩을 보고 변화 영역을 고른다
- 스타일, 헤어, 체형/자세, 피부 중 하나를 선택한다
- 현재 MVP에서 실제 시작 가능한 프로그램은 스타일이다

---

### Step 1 — 스타일 프로그램 소개 (30초)

- 카피: "사진과 옷장으로 지금 입을 조합을 정합니다"
- CTA: "스타일 시작하기"
- 여기서 처음으로 스타일 프로그램 안으로 들어간다

---

### Step 2 — 설문 3개 (1분)

질문은 3개를 넘지 않는다. 진입 장벽을 낮추는 것이 목적.

**Q1. 지금 스타일이 어때요?**
- 운동복/후드티 위주
- 청바지 + 무지 티셔츠
- 회사 유니폼/캐주얼 혼합
- 잘 모르겠어요

**Q2. 스타일을 바꾸고 싶은 계기가 있나요?**
- 소개팅 / 이성 만남
- 직장 이미지 변화
- 그냥 나 자신을 위해
- 딱히 없어요, 그냥 궁금해서

**Q3. 한 달 스타일링 예산은?**
- 5만원 이하
- 5~15만원
- 15~30만원
- 30만원 이상

> 설문 결과는 AI 피드백 프롬프트에 컨텍스트로 주입됨.

---

### Step 3 — 사진 업로드 + 옷장 확인 (2분)

- 안내 카피: "지금 입고 있는 옷 그대로 찍어주세요. 잘 나올 필요 없어요."
- 업로드 가이드: 얼굴 포함 전신 / 자연광 권장 / 정면
- 가이드 예시 이미지 제공 (잘된 예 / 안된 예)
- 촬영이 어려우면 "오늘 입은 옷 설명하기" 텍스트 대체 옵션 제공
- 사진 파일은 PNG/JPG/WEBP만 허용하고 10MB를 넘으면 업로드 전에 거부한다
- 텍스트 대체 설명은 너무 짧으면 분석 시작 조건으로 인정하지 않는다
- 옷장은 기본 화면에서 요약만 보여주고, 수정 버튼을 눌렀을 때만 편집한다
- 이전 추천 반응은 다음 분석에 조용히 반영하되, 업로드 화면 기본 상태에서는 긴 문장 목록을 펼쳐두지 않는다
- 오늘의 목표와 자신감은 기본값을 제공하고, 필요한 경우 보조 설정에서만 바꾼다
- 사이즈 정보는 보조 설정에서만 입력한다
- 사진 또는 텍스트 설명 1개 이상 + 옷장 스냅샷 1개 이상이 있어야 분석을 시작한다

이 단계가 필요한 이유:

- RE:MAN은 사진만 평가하는 서비스가 아니라 `지금 가진 옷으로 바꿀 수 있는 조합`을 제안하는 서비스다.
- 옷장 컨텍스트 없이 추천하면 유저가 바로 실행하기 어렵다.
- 목표와 자신감 선택은 기본값을 갖고 실제 선택 상태로 저장되어 AI 피드백에 반영되어야 한다.
- 설문 3개를 다시 저장할 때 기존 `style_goal`, `confidence_level` 같은 보조 코칭 기준을 덮어쓰면 안 된다.
- 설정 저장 직후 새 체크로 이동하는 빠른 라우팅에서도 보조 코칭 기준과 피하고 싶은 스타일이 비지 않도록 단기 세션 fallback을 허용한다.

---

### Step 4 — AI 분석 중 (대기 화면)

- 로딩 카피: "AI가 지훈님 스타일을 분석하고 있어요..."
- 예상 시간: 10~20초
- 이탈 방지용 진행 텍스트 노출:
  - "핏을 분석하는 중..."
  - "컬러 밸런스 확인 중..."
  - "개선 포인트 정리 중..."

---

### Step 5 — 스타일 체크 세션 결과

피드백 구조 (항상 이 포맷 유지):

**① 현재 스타일 진단 (1~2줄)**
> "지금 코디는 편안함에 집중되어 있고, 핏보다 기능성 위주예요."

**② 개선 포인트 3가지**
> 1. 바지 핏 — 일자핏으로만 가면 더 깔끔해 보여요
> 2. 상의 레이어링 — 티셔츠 위에 얇은 자켓 하나만 더해도 달라져요
> 3. 신발 — 뉴발 말고 슬립온 계열로 바꾸면 전체 무드가 가벼워져요

**③ 지금 바로 할 것 (1가지만)**
> "오늘 입은 조합에서 상의만 더 정돈된 색으로 바꿔보세요."

**④ 추천 조합 1개**
> "무지 티셔츠 + 일자핏 청바지 + 흰색 스니커즈. 지금 가진 옷 안에서 가장 정돈된 인상을 만들 수 있어요."

**⑤ 실착 미리보기 준비값**
> 추천 조합은 `try_on_prompt`로도 반환할 수 있다. 단, MVP 프론트에서는 실착/레퍼런스 생성 UI를 노출하지 않는다.

실착 단계는 비용과 대기 시간이 있는 별도 기능이다.
사용자가 결과를 보는 중 다른 생성 기능이 동시에 호출되면 실패다.

---

### Step 6 — 다음 행동 선택

- 세션 결과 이후 노출 (압박 없이)
- 기본 CTA는 결과 저장 또는 홈 복귀다
- MVP 기본 선택지는 추천 반응 저장, 옷장 수정, 기록 보기, 새 스타일 체크 시작이다
- deep dive, 실착 미리보기, 크레딧은 기본 프론트에서 숨긴다
- 7일 루틴은 사용자가 명시적으로 원할 때만 보조 경로로 제공한다

---

## 성공 지표

- Step 2 완료율 > 80%
- Step 3 사진 업로드율 > 60%
- 첫 스타일 체크 결과 확인율 > 50%
- 결과 이후 추가 체크 선택율 > 20%

---

## Revisit Rule

- 처음 온 사용자는 랜딩과 기능 허브에서 시작한다
- 이미 스타일 체크 결과가 있어도 홈/스타일 CTA는 최근 결과로 보내지 않는다
- 재방문 사용자는 새 사진 체크를 시작하고, 과거 결과는 `/history`에서 확인한다
