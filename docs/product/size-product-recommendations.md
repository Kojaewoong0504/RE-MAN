# Size / Product Recommendation v1

## Purpose

RE:MEN의 사이즈/상품 추천은 사용자가 옷을 바로 구매하게 만드는 커머스 기능이 아니다.
MVP의 목적은 추천 조합을 볼 때 "내 평소 사이즈라면 어떤 후보부터 보면 되는지"를 알려주는 것이다.

## Scope

- 사용자가 직접 입력한 평소 사이즈를 기준으로 한다.
- 전신 사진에서 신체 치수나 의상 실측을 추정했다고 표현하지 않는다.
- 외부 커머스 카탈로그, 실시간 재고, 가격, 구매 링크는 v1 범위가 아니다.
- 앱 내부의 작은 기준 카탈로그에서 카테고리/사이즈가 맞는 후보를 보여준다.
- 추천 결과는 "구매 추천"이 아니라 "카탈로그 후보"와 "사이즈 체크 후보"로 표시한다.

## Input

```json
{
  "height_cm": "175",
  "weight_kg": "72",
  "top_size": "L",
  "bottom_size": "32",
  "shoe_size_mm": "270",
  "fit_preference": "너무 붙지 않는 레귤러핏"
}
```

모든 필드는 선택 입력이다.
단, 사이즈 후보를 만들려면 `top_size`, `bottom_size`, `shoe_size_mm`, `height_cm`, `weight_kg` 중 하나 이상이 있어야 한다.

## Output

결과 화면은 추천 조합 아래에 아래 정보를 표시한다.

- 기준: 사용자가 직접 입력한 평소 사이즈
- 카테고리: 상의, 하의, 신발 중 추천 조합에 해당하는 항목
- 사이즈 후보: 예: `L`, `32`, `270`
- 확인 포인트: 기장, 허리, 발볼처럼 사용자가 실제로 확인해야 할 한 가지
- 내 옷장 기준: 같은 카테고리의 등록 아이템에 사이즈/착용감이 있으면 참고 근거로 표시
- 내부 카탈로그 후보: 카테고리, 기준 사이즈, 왜 먼저 볼 만한지

결과 화면 기본 UI에서는 이 섹션을 `사이즈 체크 후보`로 표시한다.
가격, 구매 버튼, 외부 판매 링크는 노출하지 않는다.
사이즈 후보가 없을 때도 섹션을 숨기지 말고, 설정의 `#size-profile` 영역에서 평소 사이즈를 추가하면 다음 결과부터 후보를 좁힐 수 있다고 안내한다.

## Non-Negotiables

- 사진만 보고 "키/몸무게/치수/사이즈를 파악했다"고 말하지 않는다.
- 외부 판매 중 상품처럼 보이는 UI, 가격, 구매 버튼을 만들지 않는다.
- 사이즈가 없으면 "사이즈 정보를 추가하면 후보를 좁힐 수 있다"고 안내한다.
- 옷장 아이템의 사이즈/착용감 근거는 실제 착용 참고 정보로만 표시하고, 치수 추정처럼 표현하지 않는다.
- 실제 상품 카탈로그 연동 전까지는 `reference`, `candidate`, `internal catalog` 표현만 사용한다.
- 내부 카탈로그 후보를 실제 재고 또는 구매 가능 상품으로 표현하지 않는다.

## Future

- 사용자 치수 입력 고도화
- 옷장 아이템별 사이즈와 착용감 기록
- 실제 상품 카탈로그, 가격, 재고, 구매 링크 연동
- 반품/교환 리스크를 줄이는 사이즈 confidence score

## Next Step

사이즈 후보 v1은 옷장 아이템의 `size`, `wear_state`, `condition`을 함께 참고한다.
이 정보는 상품 추천이 아니라, "내 옷장에서 비슷한 조합을 다시 고를 때 어떤 사이즈가 잘 맞았는지"를 기억하기 위한 용도다.
